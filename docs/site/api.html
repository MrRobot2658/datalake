API & 数据表

SQL Engine / ID-Mapping / Assistant 三服务完整接口目录（254+ 接口，24 个域）+ 调用约定 + 核心数据表。

调用约定

控制台全部经 Nginx 网关 :8080 访问；SQL Engine 的接口统一挂在 /api/* 前缀下（网关剥离 /api 后转发）。

服务网关路径直连端口
SQL Engine（CDP 主体）/api/*:8002
ID-Mapping（OneID）/ID-Mapping/*:8001
智能助手 Assistant/assistant/*:8004
  • 多租户：几乎所有接口都需 tenant_id（路径参数或请求体），按租户强制隔离。
  • 鉴权：先 POST /api/auth/login 拿 token，后续请求携带；网关在 http 层统一注入鉴权头。
  • 安全边界：圈人 / 查询永不直出 SQL —— 走模板（/api/query/*）或 DSL 校验链（/api/dsl/*）。
  • Swagger：SQL Engine · ID-Mapping（在线交互式文档，含请求/响应模型）。
鉴权 Auth查询与模板 Query圈人 DSL（安全边界）自然语言圈人 & 受众群组 Groups标签 Tags对象 ObjectsETL 导入统一 Unify连接 Connections调度 Scheduler（Airflow）触达 Engage客户 Accounts分析 Analyst语义层 Semantic知识库 Knowledge应用 Apps监控 Monitor协议 Protocols隐私 Privacy平台 Platform设置 / IAMID-Mapping 服务智能助手 Assistant

快速上手示例

# 1) 登录拿 token
curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@acme.com","password":"demo123"}'

# 2) 模板查询：按 OneID 查画像宽表
curl -X POST http://localhost:8080/api/query/profile_by_one_id \
  -H "Content-Type: application/json" \
  -d '{"params":{"tenant_id":1001,"one_id":100001}}'

# 3) 自然语言查询
curl -X POST http://localhost:8080/api/nl-query \
  -H "Content-Type: application/json" \
  -d '{"question":"查手机号13800138001的用户画像","tenant_id":1001}'

# 4) DSL 圈人：校验 → 预估人数（不执行 SQL）
curl -X POST http://localhost:8080/api/dsl/estimate \
  -H "Content-Type: application/json" \
  -d '{"tenant_id":1001,"rule":{"object":"user","logic":"AND","conditions":[],"relations":[]}}'

# 5) 自然语言圈人 → 候选 DSL（已过校验层）
curl -X POST http://localhost:8080/api/agent/segment/draft \
  -H "Content-Type: application/json" \
  -d '{"question":"近30天下过单且客单价大于500的用户","tenant_id":1001}'

# 6) 渠道 → OneID 映射 / 合并日志（ID-Mapping）
curl http://localhost:8080/ID-Mapping/mapping/1001/phone/13900001111
curl http://localhost:8080/ID-Mapping/merge-log/1001
