Quasar — 为 Agent 提供数据底座

多数据源计算存储智能代理：把散落各处的数据（数据库 / 数仓 / 数据湖 / 对象存储 / 消息 / API / 文件）统一接入、自动清洗、灵活转化、高性能计算，以 AI Agent 为自然语言入口，向上对 Agent 暴露结构化数据接口（OneID 画像宽表 / 圈人结果 / 聚合指标）。默认存储引擎 Apache Doris，支持计算存储分离与一体化部署。

🧱 1 平台底座 + 9 业务模块 + 3 扩展菜单 🔌 44 个主流连接器 🤖 41 个只读 MCP 工具 ⚡ 实时 OneID 归一 🪄 NL 圈人 / 生成看板 🌬️ 真实 Airflow 调度

产品界面

登录即用的对话式控制台：左侧会话 · 中间智能助手（Ask / Agent）· 右侧数据底座 + 知识库。

5 分钟跑起来

本地全栈即跑、登录即用。一条 docker compose 起全部组件 + 前端生产构建。

# 0. 克隆
git clone https://github.com/MrRobot2658/Quasar.git
cd Quasar

# 1. 启动全栈（含前端构建）
docker compose up -d --build
docker compose ps                       # 等待全部 healthy
bash scripts/apply_migrations.sh        # 应用增量迁移 + 演示数据

# 2. 灌入演示数据（多渠道合并模拟）
bash scripts/simulate_kafka.sh          # 走完整 Kafka 链路（含租户 1001/1002）

# 3. 打开控制台并登录
#    http://localhost:8080/console/  →  admin@acme.com / demo123
详细安装选项（DeepSeek 配置 / 接真实 Doris / 规模模拟 / 访问入口 / 使用规则）见 安装与使用；整体数据链路与组件端口见 架构总览。
