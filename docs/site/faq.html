常见问答 FAQ

关于产品定位、部署依赖、数据安全、接入能力与规模扩展的常见问题。

产品定位

Q · Quasar 和传统 CDP / BI 有什么区别？

传统平台只有 SQL 层——你问什么它查什么，回答精确但孤立。Quasar 内建语义融合层：以「实体 + 概念 + 关系」建模，把数据库（结构化）和知识库（非结构化）双源融合，Agent 不只给数字，还能匹配口径、关联上下文、构建归因、生成建议。从「查询工具」升级为「业务伙伴」。

Q · Quasar 就是又一个套壳的 NL2SQL 吗？

不是。NL2SQL 只解决「会不会写 SQL」，Quasar 解决「懂不懂业务」。而且LLM 绝不直出 SQL：候选 DSL 必须过 校验层（字段/操作符/关系/跳数）才会被展示或执行，安全可控。

部署与依赖

Q · 必须上 Doris / Flink 吗？能单机跑吗？

能。支持一体化部署（单机开箱即用）与计算存储分离（Doris 存算分离 + 计算弹性扩缩）两种形态。本地 dev 一条 docker compose up 起全栈，用轻量组件模拟生产的 Flink/Doris。详见 安装与使用。

Q · 没有 DeepSeek Key 能用吗？

能。自然语言圈人/查询会降级到规则模式（AGENT_LLM_ENABLED=0），其余真实功能（数据源导入、画像检索、DSL 圈人、看板）不受影响。配置 Key 后即可启用 NL 圈人/生成看板/智能助手。

Q · 语义层和知识库需要额外部署吗？

不需要。两者都是默认功能：知识库随栈启动（文件存 kb_data 卷、元数据落库）；语义层构建在对象模型 + 知识库之上，无需单独的服务。

Q · 怎么接真实 Doris？

设环境变量 OLAP_BACKEND=doris OLAP_HOST=doris-fe OLAP_PORT=9030 即可。存储经 executor.py 解耦，平台其余部分无需改动。

数据安全与治理

Q · LLM 会直接生成并执行 SQL 吗？

不会。所有查询走 engine.py 模板（named template + 参数绑定）或 dsl.py 的 validate → compile 路径。LLM 只产出候选 DSL，必经校验层；这是查询构建的安全边界。

Q · 多租户怎么隔离？

所有查询按 tenant_id 强制隔离，顶栏 Workspace 切换租户（1001/1002）。登录后 workspace 自动切到该用户所属租户。

Q · 有权限和审计吗？

有。设置提供 IAM 成员/角色/团队、API 令牌、审计日志（Audit Trail）和 MCP 工具清单，均为真实功能。

数据接入与能力

Q · 支持哪些数据源？

数据源目录平铺 44 个主流连接器，按 8 类分组：数据库/数仓/数据湖/查询引擎/对象存储/流/文件/API（MySQL、PostgreSQL、ClickHouse、MongoDB、Snowflake、BigQuery、Iceberg、Delta、S3、Kafka、Trino…）。CSV/粘贴可直接走可视化 ETL 导入。

Q · 可视化编排是真调度还是模拟？

真调度。「管道」执行经 Apache Airflow REST API 触发参数化通用 DAG，用动态任务映射（.expand）按画布节点展开多任务。Airflow 不可达时优雅降级为本地模拟，不报错。

Q · 圈人结果怎么激活到下游？

存为受众后，通过应用市场连接 CRM/广告/消息/分析等下游（Salesforce、广点通、短信、企微、神策等），状态按租户持久化。

Q · 智能助手能做什么？

右上角智能助手是多智能体：数据查询（41 个只读 MCP 工具）、分析（建图表/看板）、任务（后台运行）、通用导航；并有主动式埋点 Copilot 主动给建议。

规模与扩展

Q · 能支撑多大数据量？

用 bash scripts/scale-up.sh {{dev|medium|large|xlarge}} 按规模模拟集群（<1 万 / 1 亿 / 2 亿）。生产形态下计算弹性扩缩 + Doris 存算分离支撑大体量；大租户独立库 + BE Tag，小租户共享库按 tenant_id 分区。

Q · 和现有数仓/BI 冲突吗？

不冲突。Quasar 可作为语义层与画像中枢叠加在现有数仓之上（连数仓、Reverse ETL 回流），向上为 Agent/MA/BI 提供结构化数据接口。
